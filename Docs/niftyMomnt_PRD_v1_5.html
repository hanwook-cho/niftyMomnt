1. Overview & Scope

1.1 Purpose of This Document

This Product Requirements Document (PRD) defines the functional behavior, UX logic, technical architecture, and feature acceptance criteria for niftyMomnt v1.0. It is the primary reference for design, engineering, and QA. It builds directly on the market and user research established in MRD v1.1.

1.2 Product Summary

niftyMomnt is a moments studio for iOS. It captures five distinct asset types through a unified Capture Hub, applies on-device intelligence to tag and cluster moments automatically, and enables users to shape those moments into stories — for public sharing, trusted-circle sharing, or private keeping.

Design principle: Every interaction should feel like the app understood what you wanted before you did.

1.4 Design Constraints

  • Platform: iOS 26.0 minimum. iPhone 15 and above. Required for updated AI Intelligence frameworks, newly updated SDK APIs, AVFoundation AVCaptureMultiCamSession, Neural Engine performance, Journaling Suggestions API, and computational photography APIs. No iPadOS in v1.0. UPDATED v1.5
  • Offline-first — hard requirement: The app must be fully functional with zero network connectivity. This includes capture of all 5 asset types, on-device indexing, journal browsing, Reel assembly, preset rendering, Sound Stamp acoustic analysis, and AI Nudge Engine. The only features requiring connectivity are Lab cloud processing (opt-in) and trusted-circle link delivery.
  • Privacy non-negotiable: Private vault content must be end-to-end encrypted. Never used for AI training. Never surfaced in community feed. Never watermarked. Dual-camera secondary capture and Sound Stamp audio are processed on-device only and never stored as viewable or playable files.
  • Private by design — brand claim: niftyMomnt contains no third-party trackers, no third-party analytics SDKs, and no advertising networks.
  • On-device first: All high-frequency operations run on-device. Cloud ("Lab") is opt-in only, ephemeral, and explicitly user-triggered.
  • Performance target: Capture-to-preview latency under 300ms. Cold launch under 1.5s. Journal load under 2s for up to 5,000 assets. Sound Stamp acoustic analysis completes within 300ms of buffer end — does not block UI.

2. Asset Taxonomy — The Five Moment Types

2.1 Overview

niftyMomnt defines five discrete asset types — each designed to capture a different dimension of a moment. Together they form a complete sensory vocabulary: image, motion, time, sound, and atmosphere. Every asset type feeds the same downstream pipeline: on-device indexing, vibe tagging, journal organization, and story assembly.

Design principle: The asset type should feel like a natural extension of what the moment calls for — not a mode the user has to consciously choose.

Non-Destructive Derivative Assets NEW v1.5

The Fix feature (§3.8) introduces a derivative asset state for Still and Live assets. When a user applies a Fix (crop, rotate, or flip), the system writes a derived version alongside the original in the vault. The original is never modified or deleted. Key rules governing derivatives:

  • The derivative is the primary displayed version in the journal, Moment detail, widgets, and share flows.
  • The original is retained silently — always recoverable via "Revert to original."
  • Derivatives are stored with a fix_applied: true flag and a reference to the source asset ID in the intelligence graph.
  • Lab processing and Enhanced AI always operate on the derivative if one exists — never on the original unless the derivative has been reverted.
  • Derivatives inherit all vibe tags, acoustic tags, and ambient metadata from the original asset. Fix does not re-trigger indexing.
  • A derivative does not count as a new asset for Roll Mode or streak calculations.

2.4 Asset Interoperability & Export

  • Still: Exports as HEIF (default) or JPEG. When a derivative exists (Fix applied), the derivative is exported by default. Watermark applied on free tier.
  • Live: Exports as Apple Live Photo (HEIF + MOV pair). Fix applies to the HEIF component only — MOV exports at original dimensions. Falls back to Still in non-Apple contexts.
  • Clip: Exports as MP4 (H.264). Compatible with Instagram Reels, TikTok, and all major platforms.
  • Echo: Exports as M4A audio file. Voice-to-Prose engine generates three rewrite variants from transcript.
  • Atmosphere: Exports as MP4 (still + audio looped) for social sharing. Alternatively as HEIF + M4A pair for archive.

2.5 Dual-Camera Contextual Capture

When the user captures a Still or Live asset, the secondary camera fires within 50ms. The secondary image is never shown, stored, or exported — it is processed on-device to extract contextual metadata and discarded. Full specification unchanged from PRD v1.4 §2.5.

New in v1.5

2.6 Sound Stamp — Ambient Audio for Still Captures

Privacy principle: Sound Stamp audio is never written to disk under any mode. The 1.5s buffer exists only in memory, passes through the on-device acoustic classifier (same pipeline as §4.3), and is released. The audio is metadata infrastructure, not an asset. It is never stored, never exported, and never transmitted — including under Mode 1 (Enhanced AI) and Mode 2 (Lab).

2.6.1 Overview

Sound Stamp is an opt-in feature that captures a 1.5-second ambient audio buffer during every Still capture — 0.5s pre-roll from an always-on microphone buffer (same buffering model as Live Photo) plus 1.0s post-shutter. The buffer is analysed on-device for acoustic vibe tags identical to those defined in §4.3, then immediately discarded. No audio file is ever created.

Sound Stamp closes the acoustic tagging gap for Still-only users. Prior to v1.5, §4.3 Acoustic Signature Analysis applied only to Echo, Clip, and Atmosphere assets. A user who primarily captures Stills received zero acoustic vibe enrichment. Sound Stamp extends acoustic tagging to every Still capture without requiring mode switching or user decision.

2.6.4 Permission & Opt-In Model

  • Default: Off. Sound Stamp is never active on first install without explicit user action.
  • Onboarding: Offered in Step 5 (Optional enrichments) as "Sound Stamp" with plain-language explanation: "Capture 1.5 seconds of ambient sound with every photo — niftyMomnt uses it to tag the vibe and immediately discards the audio. Nothing is stored or shared."
  • Settings → Capture → Sound Stamp: Toggle (default: Off). Sub-label: "1.5s ambient audio at capture. Analysed on-device, never stored." Expandable "Learn more" explains what is and isn't retained.
  • Microphone permission: NSMicrophoneUsageDescription is already requested in Onboarding Step 3 for Echo and Atmosphere capture. Sound Stamp does not require a new permission — it uses the existing grant. If microphone permission is denied, Sound Stamp is silently unavailable (toggle is greyed out in Settings with a note: "Requires microphone access — enable in iOS Settings.").

2.6.5 Naming Discipline

"Sound Stamp" is the only permitted name for this feature in all user-facing surfaces. Never use "ambient recording," "microphone capture," "audio sample," or similar phrasing that implies stored content. The name signals brevity and output type — a stamp is a quick impression, not a file.

3. Capture Hub — UX & Interaction Design

3.1 Capture Hub Overview

The Capture Hub is the primary screen and heart of niftyMomnt. It is a full-bleed viewfinder with minimal chrome, a gesture-driven mode navigation system, and a smart shutter that adapts its behavior to the active asset type. The UI disappears into the moment — controls surface only when needed. All overlay surfaces use iOS 26 Liquid Glass materials (UIVisualEffectView with .systemUltraThinMaterialDark).

When the mode changes, the mode name renders center-frame in spaced capital letters at 10% opacity and fades to 0% over exactly 500ms. 3.5 Additional Capture Hub UX Rules UPDATED v1.5

  • Self-timer: Accessible via top bar. Options: Off (default), 3s, 10s. When active, a countdown ring overlays the shutter and a large center countdown number appears in the viewfinder. Both primary and secondary cameras — and Sound Stamp if enabled — fire simultaneously at countdown completion.

  • Sound Stamp indicator: When Sound Stamp is enabled, a subtle ≋ icon appears inside the shutter button in Still mode only, in the active preset accent color at 70% opacity. Post-capture, the three-arc pulse animation fires (§2.6). No indicator in other modes.
  • Camera flip: Double-tap anywhere on the viewfinder (not the shutter row) to flip between rear and front camera. No flip button in the shutter row.
  • Accessibility: All gesture interactions have equivalent button controls accessible via AssistiveTouch. Haptic feedback is mandatory for all capture events.
  • Orientation: Portrait primary. Landscape supported for Clip and Atmosphere.

3.8 Still Photo Fix — Non-Destructive Geometry Corrections

Scope principle: Fix is a correction tool, not a creative editor. It is intentionally limited to geometry (crop, rotate, flip). It does not expose brightness, contrast, saturation, healing, or any other adjustment. The word "Edit" must never appear in this flow — all UI uses "Fix" (verb) and "Fixed" (past state).

3.8.1 Overview

Fix allows users to apply non-destructive geometry corrections to Still and Live assets post-capture, without leaving the app. It is available on Still and Live assets only — not Clip, Echo, or Atmosphere. All corrections are non-destructive: the original asset is always preserved in the vault, and a derivative is written alongside it (§2.1 Non-Destructive Derivative Assets).

4. On-Device Intelligence — Cold Indexing

4.1 Overview

All high-frequency intelligence runs entirely on-device using Apple's Core ML, Vision, and AVFoundation frameworks. This is the "Cold" indexing layer — it runs in the background after capture, silently building a rich metadata graph for every asset without any network call. The results feed the journal, vibe tagging system, moment clustering, and story assembly engine.

Privacy principle: Cold indexing never leaves the device. No asset or metadata is transmitted in this layer. Cloud processing is opt-in only and handled by the separate Hybrid Lab system (§5).

4.2 Image Classification & Categorization

Applied to: Still, Live, Clip (per frame), Atmosphere (still component). On-device Vision classifier identifies scene context and assigns place tags and vibe tags. Full taxonomy unchanged from PRD v1.4 §4.2.

4.3 Acoustic Signature Analysis UPDATED v1.5

Applied to: Still (via Sound Stamp, opt-in), Echo, Clip (audio track), Atmosphere (audio component). Background audio analysis runs after capture using Core ML audio classification. Results populate acoustic vibe tags that enrich the moment's metadata graph.

Sound Stamp integration: when Sound Stamp is enabled (§2.6), Still assets now enter the acoustic analysis pipeline using their 1.5s ambient buffer. The same Core ML model and tag taxonomy is used for all asset types — Still/Sound Stamp results are not differentiated from Echo/Clip results in the intelligence graph. A Still asset with Sound Stamp tags is acoustically equivalent to an Echo asset with the same tags for all downstream features (journal, Reel assembly, Mood Map, Annual Review).

Applied to: Still, Live, Clip (sampled frames), Atmosphere (still component). Dominant color palette extraction (up to 5 HSL values) for vibe tag enrichment, preset recommendation, and visual clustering. Unchanged from PRD v1.4 §4.4.

4.5 Ambient Metadata Harvesting

Applied to all asset types at moment of capture. WeatherKit, Core Location (elevation), system clock (sun position), MusicKit (Now Playing), Spotify iOS SDK. Unchanged from PRD v1.4 §4.5.

4.6 Moment Clustering

GPS proximity (200m radius), time proximity (90-minute window), vibe similarity, and asset type mix drive clustering. Runs on-device when charging and idle. Unchanged from PRD v1.4 §4.6.

Fix operations are entirely on-device and do not involve any AI mode. Full mode specifications unchanged from PRD v1.4 §5.

6. Story Engine & AI Reel Assembler

The Story Engine transforms raw Moment clusters into publishable content. Fix-corrected assets (derivatives) are used in Reel assembly and journal hero selection — the system always works from the primary displayed version of each asset. Sound Stamp acoustic tags from Still assets contribute to Reel sequencing logic and music matching in the same way Echo acoustic tags do. All other Story Engine specifications unchanged from PRD v1.4 §6.

7. AI Nudge Engine — Post-Capture Reflection

The AI Nudge Engine surfaces contextual reflection cards post-capture. Nudge question tone and chip copy updated for MZ audience per UI/UX Spec v1.6 §6.3 — questions are phrased in aesthetic/felt register, not analytical register. Sound Stamp acoustic tags from Still assets feed the Nudge Engine trigger logic (Acoustic signal trigger pattern) in the same way Echo acoustic tags do. All other Nudge Engine specifications unchanged from PRD v1.4 §7.

8. Private Vault & Trusted Sharing

The Private Vault is the user's encrypted personal archive. Fix derivatives are stored in the vault alongside their originals — both are encrypted with the same AES-256 key. The original is never exposed or deleted by the Fix operation. All other vault and trusted sharing specifications unchanged from PRD v1.4 §8.

9. Vibe Preset System UPDATED v1.5

9.1 Overview

The vibe preset system is niftyMomnt's primary aesthetic differentiation. Presets are applied at capture (real-time viewfinder rendering) and persist through all downstream outputs. A preset is not just a color filter — it defines a full aesthetic character including grain simulation, tone mapping, chromatic shifts, and UI accent color.

9.3 Preset Selection UX UPDATED v1.5

The always-visible five-circle strip from previous versions is replaced by the preset bar model in v1.5:

  • Default state: Active preset name shown in spaced caps with 5pt accent dot in the preset bar. Swipe left/right on the bar to cycle through presets one at a time.

10.1 Capture Hub

US-01: Mode Navigation UPDATED v1.5

As a user, I want to switch between capture modes using a natural swipe gesture and cycle presets with a dedicated bar, so that I can move fluidly through modes and aesthetics without breaking my focus on the moment.

ACAcceptance Criteria
1Swiping left on the lower third of the viewfinder advances the mode in sequence: Still → Live → Clip → Echo → Atmosphere → Still.
2Swiping right reverses the sequence. Wraps handled gracefully in both directions.
3The persistent mode anchor (mode name at 28% opacity + three dot indicators) is always visible at the base of the viewfinder. The ghost mode label flashes at 10% opacity center-frame for exactly 500ms on mode change then fades to 0%.
4Mode transition completes within 150ms of gesture recognition.
5A fast swipe (velocity >800pt/s) advances only one mode.
6 NEWSwiping left/right on the preset bar cycles presets one at a time. Preset name, accent dot, 2pt top strip, shutter ring color, and Dynamic Island dot all update in a single 120ms crossfade.
7 NEWLong-pressing the preset bar expands a full preset picker overlay from the bottom. All installed presets are shown as large swatches with names and vibe descriptors. Tapping a preset selects it and dismisses the overlay. Swiping down dismisses without change.
8 NEWDouble-tapping anywhere on the viewfinder (not the shutter row) flips the camera (rear ↔ front) with a 200ms mirror transition. No flip button exists in the shutter row. A one-time ghost hint "double-tap to flip" is shown on first use only, never again.

10.5 Still Photo Fix

US-08: Still Photo Fix — Non-Destructive Geometry Corrections

As a user, I want to quickly correct the crop, rotation, or flip of a photo I captured without leaving the app, knowing my original is always preserved.

ACAcceptance Criteria
1Fix is available on Still and Live assets only. It does not appear in context menus or full-screen viewers for Clip, Echo, or Atmosphere assets.
2Fix is accessible from two entry points: (a) long-press on asset in Moment detail grid → context menu includes "Fix"; (b) full-screen asset viewer → "Fix" pill button bottom-right.
3The Fix modal opens as a full-screen push from the right. The top bar shows: Cancel (left) · Fix (center title) · Done in purple (right). The word "Edit" never appears anywhere in this flow.
4Crop: user can drag any edge or corner handle to crop. Dimmed overlay shows excluded area in real time. Rule-of-thirds grid is always visible. Double-tapping the photo resets crop to full frame. Minimum crop size: 200×200pt. Aspect ratio is free.
5Rotate 90°: single tap on Rotate tool rotates the photo 90° clockwise with a spring animation (200ms). Repeatable. Crop frame auto-adjusts to new orientation.
6Flip: single tap on Flip tool reveals an inline sub-row with Horizontal and Vertical options. Each option applies instantly with a mirror animation (150ms). Both can be active simultaneously. Flip state compounds correctly with rotation.
7Tapping Done with changes applied writes a derivative asset to the vault alongside the original. The original is never modified. A "Fixed" badge appears on the asset thumbnail in the Moment grid.
8Tapping Done with no changes made is identical to Cancel — no derivative is written.
9"Revert to original" is always visible in the Fix modal and accessible from the "Fixed" badge in the Moment grid. Revert shows confirmation: "Remove fix? Your original photo will be shown." Confirming deletes the derivative and restores the original as primary.
10For Live assets: Fix applies to the HEIF still frame only. The MOV motion component is not affected. A one-line disclosure "Motion clip not affected" is shown in the Fix tool bar when a Live asset is open.
11Fix derivatives inherit all vibe tags, acoustic tags, and ambient metadata from the original. Fix does not re-trigger indexing. A derivative does not count as a new asset for Roll Mode or streak calculations.
New in v1.5

10.6 Sound Stamp

US-09: Sound Stamp — Ambient Acoustic Tagging for Still Captures

As a user with Sound Stamp enabled, I want my Still photos to be automatically tagged with the ambient sound of the moment — so my journal reflects the full sensory character of where I was, without any extra effort.

ACAcceptance Criteria
1Sound Stamp is Off by default. It is offered during onboarding Step 5 (Optional enrichments) and in Settings → Capture → Sound Stamp. It cannot be activated by any other means.
2When Sound Stamp is enabled, a subtle ≋ icon appears inside the shutter button in Still mode only (active preset accent color, 70% opacity). No icon appears in other capture modes.
3When a Still is captured with Sound Stamp enabled, a 1.5s ambient audio buffer is captured (0.5s pre-roll + 1.0s post-shutter). The buffer is analysed on-device by the same Core ML acoustic classifier used for Echo/Clip/Atmosphere assets.
4The audio buffer is never written to disk, never stored as a file, never exported, and never transmitted under any AI mode (Mode 0, 1, or 2). Maximum in-memory lifetime: ~500ms.
5Post-capture, three concentric arcs pulse outward from the shutter ring in the active preset accent color for 1.5s. Amplitude is fixed (not reactive to audio level). Under Reduce Motion: a static arc at 30% opacity appears for 1s then fades.
6Acoustic tags produced by Sound Stamp are written to the asset's metadata within 300ms of buffer completion, without blocking the UI. Tags appear in the Moment detail vibe panel — identical appearance to acoustically-tagged Echo assets.
7If microphone permission is denied, Sound Stamp is unavailable. The Settings toggle is greyed out with a note: "Requires microphone access — enable in iOS Settings." Sound Stamp is never silently inactive — its state is always clearly communicated.
8Sound Stamp acoustic tags from Still assets feed the Nudge Engine, Reel assembler, Mood Map, and Annual Review in the same way Echo acoustic tags do. No differentiation in downstream features between Still/Sound Stamp tags and Echo tags.

11.2 Privacy & Security

  • Private vault encryption: AES-256. Key stored in iOS Keychain. Key never transmitted. Fix derivatives encrypted with the same key as their source original.
  • Mode 0 — on-device: No network calls. Sound Stamp audio never leaves device memory. Fix operations entirely on-device. Verifiable via network inspection.
  • Mode 1 — Enhanced AI (text-only): Only anonymized text transmitted. Photos, audio files, Sound Stamp buffers, and Fix derivatives are never sent. TLS 1.3 in transit.
  • Mode 2 — Lab (encrypted visual): Assets AES-256 encrypted on-device before transmission. Sound Stamp buffers are never included — they are discarded before any mode decision. TLS 1.3. Certificate pinning. Ephemeral sandbox. Per-session explicit consent every time.
  • Sound Stamp: Audio buffer never written to disk under any condition. Maximum in-memory lifetime ~500ms. Never transmitted. iOS orange mic indicator appears — this is correct and required behavior, not a bug.
  • Face detection: On-device only. No biometric data stored or transmitted.
  • GDPR / CCPA: User can export all personal data and permanently delete all account data from Settings. Fix derivatives included in data export. Sound Stamp produces no exportable data — only the resulting tags, which are included in the standard metadata export.

11.3 Accessibility

  • VoiceOver — Capture Hub: Mode switch announces current mode. Sound Stamp state announced: "Sound Stamp active" when ≋ icon is present. Shutter announces: "Capture [mode name]. Sound Stamp will capture ambient audio." when Sound Stamp is on.
  • VoiceOver — Fix modal: Crop handles have labels: "Top edge crop handle," "Bottom-right corner crop handle," etc. Rotate announces: "Rotated 90 degrees clockwise. Total rotation: [n] degrees." Flip announces: "Flipped horizontally" / "Flipped vertically."
  • Dynamic Type: All text supports Dynamic Type up to Accessibility Extra Large. Fix modal tool bar labels scale with Dynamic Type.
  • Reduced Motion: Sound Stamp arc pulse: static arc at 30% opacity for 1s then fades. Ghost label: static fade. Capture ring: static arc. Fix animations: crossfades replace slide/spring animations.
  • Colour independence: No UI state communicated by color alone. Sound Stamp active state: ≋ icon (not just accent color). Fix "Fixed" badge: text label (not just color).
  • Gesture alternatives: All swipe-based gestures have AssistiveTouch button equivalents. Double-tap flip has an equivalent button accessible from the accessibility menu.
