1. Design Principles

Every UI decision in niftyMomnt is anchored to four principles derived from the MRD user research and the PRD product philosophy.

#PrincipleWhat it means in practice
P-01Capture-first, alwaysOpen app = camera ready, every time. Zero taps between intent and capture. The viewfinder is never more than one gesture away from any screen.
P-02Feel effortless, look unfilteredUI chrome is minimal — it disappears during capture. Presets are always one tap away. The aesthetic feels like it happened naturally, not like it was engineered.
P-03Private by default, visible when chosenNo sharing action is ever automatic. Every share is an explicit choice. The vault is the default state. Privacy UI is clear, never buried.
P-04Moments over metadataTags, timestamps, and GPS are always secondary to the emotional character of the moment. The Film Archive shows vibes and feelings first — raw data is one tap deeper.

2.4 Spacing & Radius

Base unit: 4px. All spacing values are multiples of 4. Corner radius: 12px for cards, 8px for chips and badges, 50% for circular elements (shutter, avatar). Safe area insets respected on all screens — no interactive element placed behind notch or home indicator.

2.5 Motion Principles

  • Duration: micro-interactions 120–200ms; screen transitions 280–350ms; ghost label fade exactly 500ms.
  • Easing: spring(response: 0.35, dampingFraction: 0.72) for all positional animations. Linear for continuous (ring fill, waveform). EaseOut for fade-outs.
  • Reduced Motion: all animations respect UIAccessibility.isReduceMotionEnabled. Crossfades replace slides; static ring replaces animated fill.

3. Information Architecture

3.1 Navigation Model

niftyMomnt uses a capture-first, swipe-up navigation model. The app opens directly to the Capture Hub viewfinder. Swiping up from anywhere on the capture screen reveals the Film Archive — the primary secondary surface. The Film Archive contains a bottom tab bar for navigating between Film, Vault, and Settings. The Capture Hub is always one swipe down from the Film Archive. The Film feed uses a dark editorial aesthetic consistent with the capture surface.

4. Capture Hub

4.1 Screen Layout

The Capture Hub is a full-bleed dark viewfinder with four UI zones, redesigned for iOS 26 and MZ users. The old five-zone layout (top bar / viewfinder / preset strip / mode zone / shutter row) has been consolidated: the preset strip is replaced by a single active-preset bar, the mode swipe zone is elevated into a persistent bottom anchor, and the shutter row is simplified. All overlay surfaces use iOS 26 Liquid Glass — backdrop-filter with calibrated blur, saturate, and brightness (see zone specs below) — instead of deprecated flat black-only frosted stacks. This gives controls depth and native feel.

iOS 26 material rule: every overlay surface in the Capture Hub — top bar, preset bar, mode anchor, shutter row — must use the Liquid Glass formula specified per zone (e.g. blur(28px) saturate(200%) brightness(1.12) on Zone A). Do not use flat rgba(0,0,0,.36) blur(20px) alone — that pattern is the deprecated iOS 17/18 frosted glass specification. Flat rgba(0,0,0,x) without the saturate+brightness treatment is prohibited on the capture surface. This is a hard engineering constraint, not a visual preference.
Figure 2 — Capture Hub v1.8. Live preview is full-bleed from just below the Dynamic Island to the top of Zone C (same vertical stacking as iOS Camera). Zone A is a 64pt Liquid Glass overlay on the preview. Icon order (left → right): Flash ① (amber, leftmost — iOS Camera parity) · Timer ② · Film Strip Counter (centre) · Live Photo ① (right) · More ② (right). AF Lock is contextual (§4.1a) — not a persistent button.
Change 1 — Preset bar replaces strip

The five-circle horizontal strip is replaced by a single active-preset name bar spanning the full width. The preset name (e.g., AMALFI) is displayed in spaced caps with a 5pt accent dot matching the preset color. A 2pt color strip runs along the top edge of the bar. Swipe left/right on the bar to cycle presets. Long-press expands a full preset picker overlay from the bottom — discovery interaction, not always-visible menu.

Change 2 — Flip moved to gesture

The flip camera button is removed from the shutter row entirely. Double-tap anywhere on the viewfinder flips the camera, matching the gesture users already know from native iOS Camera (introduced iOS 17). This frees the shutter row to breathe — only the last-capture thumbnail and the shutter remain. A ghost hint "double-tap to flip" is shown once per session for new users, then never again.

Change 3 — Liquid Glass materials

All overlay surfaces — top bar, preset bar, shutter row — use iOS 26 Liquid Glass (backdrop-filter with zone-specific blur, saturate, and brightness). They refract and blur the viewfinder content behind them, creating depth rather than blocking it. The top bar feels like it floats above the viewfinder. The shutter row feels physically present without masking the scene.

Change 4 — Persistent mode anchor

The invisible swipe zone is replaced by a persistent mode name rendered at 28% opacity at the base of the viewfinder, with three dot indicators (previous · active · next) below it. The active dot is larger and at 55% opacity. This gives the swipe gesture a visible anchor without adding real chrome — the label is decorative but also functional orientation.

4b. Dual-Camera Contextual Capture — UI

4b.1 Overview

When the user captures a Still or Live asset, the secondary camera (whichever is not primary) fires silently within 50ms. The secondary image is never shown to the user, never stored as a viewable photo, and never exported. It is processed on-device immediately to extract contextual metadata, then discarded. The UI requirement is to surface just enough to maintain trust without adding cognitive load.

Privacy-first UI rule: the dual-camera capture is metadata infrastructure, not an asset. The UI must never present it as a "hidden photo" — the indicator and tooltip together communicate exactly what it is: context for richer tagging.

4c. Sound Stamp — Ambient Audio for Still Captures

4c.1 Overview

Sound Stamp is an opt-in feature that silently records 1.5 seconds of ambient audio during every Still capture — 0.5s pre-roll from an always-on microphone buffer plus 1s post-shutter. The audio buffer is analysed on-device for acoustic vibe tags, then immediately discarded. It is never stored as a playable file, never exported, and never surfaced to the user as content. It is metadata infrastructure — identical in philosophy to the dual-camera contextual capture.

Privacy-first rule: Sound Stamp audio is never written to disk. The 1.5s buffer exists only in memory, passes through the on-device acoustic classifier, and is released. iOS's orange microphone indicator will appear briefly during capture — this is intentional and correct. It signals transparency, not a problem.
Opt-in requirement: Sound Stamp is off by default. It must be offered as a named opt-in during onboarding Step 5 and as a clearly labelled toggle in Settings → Capture. The feature name "Sound Stamp" must be used consistently — never "audio capture" or "microphone recording."

5. Film Archive

5.1 Film Archive Layout — Editorial Card Feed

The Film Archive is a reverse-chronological roll card feed on a dark editorial surface (#0F0D0B). Each Moment is one card. 6. Nudge Card

6.1 Anatomy

The Nudge card is a bottom-sheet that appears when the NudgeEngine fires a trigger. It slides up from the bottom of the screen with a spring animation. It can appear over any surface — Capture Hub, Film Archive, or Vault. The card is dismissible by swipe-down or tapping outside.

Tone principle (MZ-first): Questions must feel like a friend texting you, not a journaling app prompting you. The register is casual, specific, and aesthetic — it re-inhabits the moment rather than asking you to explain it. Questions are phrased in the language of vibes and sensation, not narrative analysis.

Writing rule for Nudge questions: if the question could appear on a therapy intake form or a UX research debrief, rewrite it. The test: would a 22-year-old send this exact phrasing to a friend in a text? If yes, it's right.
Figure 4 — Nudge card bottom-sheet. Hero thumbnail + AI question (aesthetic/felt tone) + 2–4 vibe-labeled chips + four secondary actions including "Emoji it" tertiary. Background uses dark Film Archive (#0F0D0B). One chip tap = submitted, card dismisses with spring animation.

13. Intelligence Graph & Companion App Data Layer

13.1 Overview

The intelligence graph is the on-device structured metadata layer that powers all four companion apps — Trip Planner, Annual Review, Travel Portfolio, and Shared Journey. This section covers UI requirements for surfaces that expose intelligence graph data. Raw assets remain exclusively in the niftyMomnt encrypted vault. Companion apps are read-only consumers of the graph.

Architecture principle: companion apps never display raw assets directly without an explicit per-asset user authorization prompt. The intelligence graph — vibe tags, place history, mood data — is the only data that flows across app boundaries without a prompt.
