3.9 Post-Capture Overlay UX NEW v1.6

Timing principle: The post-capture overlay must not interrupt the user's next capture. The full overlay sequence (Vibe Tag Prompt → Quick Share Pill) completes in 7 seconds total, then the overlay dismisses automatically. Users can dismiss at any point by tapping the shutter again.

3.9.1 Sequential Flow

The post-capture overlay fires immediately after every Still or Live capture. The sequence is strictly ordered — no element fires early or late:

  1. T+0ms: Capture completes. Viewfinder freezes on the captured image for 300ms (flash effect).
  2. T+300ms: Post-capture overlay renders over the frozen frame. Location Vibe Chip appears at the top (resolved from on-device sources — see §3.9.3). Vibe Tag Prompt sticker chips animate in with spring-bounce (staggered 0/58/116/174ms delays).
  3. T+4s or user tap: Vibe Tag Prompt window closes. Selected tags are written to the asset's metadata record in the intelligence graph. The Nudge Engine queues these tags for reading when it fires (see §7).
  4. T+4.3s: Quick Share Pill slides up from Zone D (spring animation). Auto-dismisses after 3 seconds if not tapped.
  5. T+7.5s: Overlay clears. Viewfinder returns to live camera. Film strip counter decrements.

3.9.2 Vibe Tag Prompt

The Vibe Tag Prompt presents four sticker chip options. Each chip is a tilted glass pill with a unique rotation angle (−2.5°, +1.8°, −1.2°, +2.2°), varied border-radius (18–28px), and a spring-bounce entrance animation (cubic-bezier(.34,1.56,.64,1), 420ms). The first chip uses the amber accent colour; remaining chips use neutral glass.

Chip content is AI-suggested based on the active preset, on-device location, time of day, and weather. The user can tap one chip to confirm the vibe tag, or swipe to dismiss without tagging. Tapping a chip plays a subtle haptic (UIImpactFeedbackGenerator .light).

Prompt label copy: "add a vibe" in uppercase, letter-spacing .10em, opacity .38 — does not compete with the chips.

3.9.3 Location Vibe Chip

The Location Vibe Chip appears at the top of the post-capture overlay (not in the live viewfinder — zero chrome impact on capture). It displays the resolved location name + contextual vibe descriptor (e.g., "Hongdae · Seoul · golden hour").

Derivation pipeline (entirely on-device, <100ms):

  • CoreLocation reverse geocoding → neighbourhood + city name
  • WeatherKit → current conditions → mapped to vibe descriptor (golden hour, rainy evening, etc.)
  • Optional: SoundAnalysis ambient classifier or Vision scene classifier (if enabled by Live Vibe Preview setting)
  • Never uses cloud geocoding or third-party location services

3.9.4 Quick Share Pill

The Quick Share Pill uses UIActivityViewController — the standard iOS share sheet. It is the MVP share mechanism. The pill is styled as a purple-tinted glass button (background: rgba(91,79,207,.24); border: 1px solid rgba(107,78,255,.48)) showing up to three close-friends avatars and the label "send to close friends."

Tapping the pill opens the system UIActivityViewController pre-populated with the captured asset. The Trusted-Circle share flow (named recipients, persistent albums) is a Post-MVP feature and is not present in v1.0.

3.9.5 Nudge Engine Sequencing

The AI Nudge Engine reads vibe tags from the intelligence graph — it does not fire during the post-capture window. The exact sequencing is:

  1. Vibe Tag Prompt fires and the user selects a tag (or the window auto-closes)
  2. Selected tags (if any) are written to the asset's metadata record
  3. The Nudge Engine fires after the post-capture window closes, reads the stored tags, and generates a reflection card prompt that incorporates the user's vibe selection

This ensures the Nudge Engine always has access to the most recent user-provided vibe signal, making its reflection prompts more contextually accurate.

7.x Post-Capture Trigger Sequencing UPDATED v1.6

The AI Nudge Engine does not fire during the post-capture overlay window. The confirmed sequencing is:

  1. Capture event fires → Post-Capture Overlay opens (§3.9)
  2. Vibe Tag Prompt is presented (T+300ms, 4-second window)
  3. User-selected vibe tags are written to the asset's metadata record in the intelligence graph
  4. Post-Capture Overlay closes (T+7.5s or on user dismissal)
  5. AI Nudge Engine fires, reads the intelligence graph including any user-provided vibe tags from step 3, and generates a reflection card prompt calibrated to those tags
Why this order matters: Firing the Nudge Engine after the user's vibe selection — not during the prompt — ensures the AI has access to the user's actual emotional signal before generating the reflection question. A Nudge card for a "golden hour" tag should feel different from one for "city night."
