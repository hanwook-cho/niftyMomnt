1. Design Principles

Every UI decision in niftyMomnt is anchored to four principles derived from the MRD user research and the PRD product philosophy.

#PrincipleWhat it means in practice
P-01Capture-first, alwaysOpen app = camera ready, every time. Zero taps between intent and capture. The viewfinder is never more than one gesture away from any screen.
P-02Feel effortless, look unfilteredUI chrome is minimal — it disappears during capture. Presets are always one tap away. The aesthetic feels like it happened naturally, not like it was engineered.
P-03Private by default, visible when chosenNo sharing action is ever automatic. Every share is an explicit choice. The vault is the default state. Privacy UI is clear, never buried.
P-04Moments over metadataTags, timestamps, and GPS are always secondary to the emotional character of the moment. The journal shows vibes and feelings first — raw data is one tap deeper.

2.4 Spacing & Radius

Base unit: 4px. All spacing values are multiples of 4. Corner radius: 12px for cards, 8px for chips and badges, 50% for circular elements (shutter, avatar). Safe area insets respected on all screens — no interactive element placed behind notch or home indicator.

2.5 Motion Principles

  • Duration: micro-interactions 120–200ms; screen transitions 280–350ms; ghost label fade exactly 500ms.
  • Easing: spring(response: 0.35, dampingFraction: 0.72) for all positional animations. Linear for continuous (ring fill, waveform). EaseOut for fade-outs.
  • Reduced Motion: all animations respect UIAccessibility.isReduceMotionEnabled. Crossfades replace slides; static ring replaces animated fill.

4. Capture Hub

4.1 Screen Layout

The Capture Hub is a full-bleed dark viewfinder with four UI zones, redesigned for iOS 26 and MZ users. The old five-zone layout (top bar / viewfinder / preset strip / mode zone / shutter row) has been consolidated: the preset strip is replaced by a single active-preset bar, the mode swipe zone is elevated into a persistent bottom anchor, and the shutter row is simplified. All overlay surfaces use iOS 26 Liquid Glass (UIVisualEffectView with .systemUltraThinMaterialDark) instead of flat black alphas — giving controls depth and native feel.

iOS 26 material rule: every overlay surface in the Capture Hub — top bar, preset bar, mode anchor, shutter row — must use .systemUltraThinMaterialDark blur material. Flat rgba(0,0,0,x) overlays are prohibited on the capture surface. This is a hard engineering constraint, not a visual preference.
Figure 2 — Redesigned Capture Hub. Four zones using iOS 26 Liquid Glass materials throughout. Preset strip replaced by single active-preset name bar (swipe to cycle, hold to expand). Mode anchor is a persistent ultra-subtle label + dot indicators at base of viewfinder. Shutter row simplified — flip moved to double-tap gesture on viewfinder. Clip ring shown at ~60% with amber tint and countdown number inside shutter.
Change 1 — Preset bar replaces strip

The five-circle horizontal strip is replaced by a single active-preset name bar spanning the full width. The preset name (e.g., AMALFI) is displayed in spaced caps with a 5pt accent dot matching the preset color. A 2pt color strip runs along the top edge of the bar. Swipe left/right on the bar to cycle presets. Long-press expands a full preset picker overlay from the bottom — discovery interaction, not always-visible menu.

Change 2 — Flip moved to gesture

The flip camera button is removed from the shutter row entirely. Double-tap anywhere on the viewfinder flips the camera, matching the gesture users already know from native iOS Camera (introduced iOS 17). This frees the shutter row to breathe — only the last-capture thumbnail and the shutter remain. A ghost hint "double-tap to flip" is shown once per session for new users, then never again.

Change 3 — Liquid Glass materials

All overlay surfaces — top bar, preset bar, shutter row — use .systemUltraThinMaterialDark. This means they refract and blur the viewfinder content behind them, creating depth rather than blocking it. The top bar feels like it floats above the viewfinder. The shutter row feels physically present without masking the scene. Paired with the dimensional radial-gradient shutter button, the capture surface now reads as iOS 26-native.

Change 4 — Persistent mode anchor

The invisible swipe zone is replaced by a persistent mode name rendered at 28% opacity at the base of the viewfinder, with three dot indicators (previous · active · next) below it. The active dot is larger and at 55% opacity. This gives the swipe gesture a visible anchor without adding real chrome — the label is decorative but also functional orientation. The ghost-label-on-switch animation is retained on top of this as a confirmation flash.

4c. Sound Stamp — Ambient Audio for Still Captures

4c.1 Overview

Sound Stamp is an opt-in feature that silently records 1.5 seconds of ambient audio during every Still capture — 0.5s pre-roll from an always-on microphone buffer (same pattern as Live Photo) plus 1s post-shutter. The audio buffer is analysed on-device for acoustic vibe tags, then immediately discarded. It is never stored as a playable file, never exported, and never surfaced to the user as content. It is metadata infrastructure — identical in philosophy to the dual-camera contextual capture.

Sound Stamp closes the acoustic tagging gap for Still-only users: currently §4.3 (Acoustic Signature Analysis) only fires on Echo, Clip, and Atmosphere assets. A user who never captures an Echo gets zero acoustic enrichment. With Sound Stamp enabled, every Still quietly inherits an acoustic vibe signature — Crowded, Rainy, Quiet, Music, Windy, Ocean — without any mode switching or deliberate action.

Privacy-first rule: Sound Stamp audio is never written to disk. The 1.5s buffer exists only in memory, passes through the on-device acoustic classifier, and is released. iOS's orange microphone indicator will appear briefly during capture — this is intentional and correct. It signals transparency, not a problem. The onboarding disclosure and Settings toggle are the trust anchors, not hiding the indicator.
Opt-in requirement: Sound Stamp is off by default. It must be offered as a named opt-in during onboarding Step 5 (Optional enrichments) and as a clearly labelled toggle in Settings → Capture. The feature name "Sound Stamp" must be used consistently — never "audio capture," "microphone recording," or similar phrasing that implies stored content.

4c.2 Shutter Feedback — Sound Stamp Active

When Sound Stamp is enabled, Still mode capture adds a single subtle feedback element that communicates acoustic capture is happening without interrupting the moment. Nothing is shown before the shutter fires — the feedback is post-capture only.

5.3 Fix Modal — Still & Live Photo Corrections

Fix is a lightweight, non-destructive correction overlay for Still and Live assets only. It is intentionally scoped to geometry corrections — crop, rotate, and flip. It is not a creative editing tool and must not be positioned as one. The word "Fix" is used everywhere in the UI; "Edit" never appears in this flow.

Non-destructive rule: the original captured asset is always preserved in the vault unchanged. Fix writes a derivative version alongside it. If the user deletes the fix, the original is restored as the primary. This is both a privacy principle and a safety net — the vault is immutable by design.
Figure 3b — Fix modal. Full-screen dark overlay with Liquid Glass top bar and tool bar. Crop active: rule-of-thirds grid, L-handle corners, dimmed outside area. Three tools: Crop (pinch-drag), Rotate 90° (single tap, repeatable), Flip (tap → H/V sub-choice). Revert link always visible. Done writes non-destructive derivative to vault.

6. Nudge Card

6.1 Anatomy

The Nudge card is a bottom-sheet that appears when the NudgeEngine fires a trigger. It slides up from the bottom of the screen with a spring animation. It can appear over any surface — Capture Hub, Journal, or Vault. The card is dismissible by swipe-down or tapping outside.

Tone principle (MZ-first): Questions must feel like a friend texting you, not a journaling app prompting you. The register is casual, specific, and aesthetic — it re-inhabits the moment rather than asking you to explain it. Questions are phrased in the language of vibes and sensation, not narrative analysis. Chips are short, colloquial, and feel like things you'd actually say — never like survey options.

Writing rule for Nudge questions: if the question could appear on a therapy intake form or a UX research debrief, rewrite it. The test: would a 22-year-old send this exact phrasing to a friend in a text? If yes, it's right.

13. Intelligence Graph & Companion App Data Layer (PRD v1.4 §12)

13.1 Overview

The intelligence graph is the on-device structured metadata layer that powers all four companion apps — Trip Planner, Annual Review, Travel Portfolio, and Shared Journey. This section covers UI requirements for surfaces that expose intelligence graph data. Raw assets remain exclusively in the niftyMomnt encrypted vault. Companion apps are read-only consumers of the graph.

Architecture principle: companion apps never display raw assets directly without an explicit per-asset user authorization prompt. The intelligence graph — vibe tags, place history, mood data — is the only data that flows across app boundaries without a prompt.

13.2 Storage Management UI

Settings → Storage shows a clear breakdown of device usage. The intelligence graph size is displayed separately to help users understand it is always small regardless of asset volume.
